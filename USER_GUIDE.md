# Руководство пользователя KatoPro

Это пошаговая инструкция по работе с сервисом генерации ежемесячных отчётных книг Excel.

Если вы запускаете проект впервые — начните с раздела [«2. Быстрый старт»](#2-быстрый-старт-для-новичка).

## 1. Что делает программа

KatoPro создаёт **чистый шаблон отчёта на новый месяц** для одной или нескольких торговых точек.

На основе уже заполненного файла за предыдущий месяц программа:

1. добавляет новый лист (например, `Май26`);
2. проставляет даты месяца;
3. выделяет выходные и праздничные дни РФ;
4. копирует формулы (выручка, остаток, суммы, средние);
5. связывает остаток и ТМ с предыдущим месяцем;
6. защищает лист, оставляя открытыми ячейки для ввода данных.

Почтовую рассылку сервис **не** выполняет — файлы сохраняются локально в папку `generated/`.

## 2. Быстрый старт (для новичка)

Этот раздел рассчитан на новичка: от установки Docker и Git до получения кода с GitHub и первой генерации отчёта.

Репозиторий проекта: [https://github.com/Xpehutta/KatoPro](https://github.com/Xpehutta/KatoPro)

### Шаг 0. Что понадобится

1. Компьютер с Windows, macOS или Linux.
2. Программа **Docker Desktop** (включает Docker и Docker Compose).
3. Программа **Git** (чтобы скачать и обновлять код с GitHub).
4. Исходный Excel-файл торговой точки с листами месяцев  
   (например, листы `Март26`, `Апрель26`).
5. Терминал:
   - **macOS / Linux** — приложение «Терминал»;
   - **Windows** — PowerShell или «Терминал Windows».

### Шаг 1. Установите Docker Desktop

1. Откройте сайт: [https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/).
2. Скачайте версию для вашей ОС и установите её.
3. Запустите **Docker Desktop** и дождитесь статуса вроде «Docker is running».
4. Проверьте установку в терминале:

```bash
docker --version
docker compose version
```

Если команды выводят номера версий — всё готово.  
Если появляется ошибка «command not found» — перезапустите терминал или переустановите Docker Desktop.

### Шаг 2. Установите Git

Проверьте, установлен ли Git:

```bash
git --version
```

Если видите номер версии (например, `git version 2.45.0`) — переходите к шагу 3.

Если Git не найден:

| ОС | Как установить |
|----|----------------|
| **Windows** | Скачайте установщик с [https://git-scm.com/download/win](https://git-scm.com/download/win), установите с настройками по умолчанию, затем откройте **новый** терминал |
| **macOS** | В терминале выполните `xcode-select --install` или установите Git с [https://git-scm.com/download/mac](https://git-scm.com/download/mac) |
| **Linux (Debian/Ubuntu)** | `sudo apt update && sudo apt install -y git` |

Снова проверьте:

```bash
git --version
```

### Шаг 3. Скачайте (clone) репозиторий с GitHub

Это **первый** раз, когда вы получаете проект на компьютер.

1. Откройте папку, куда хотите положить проект (например, `Documents` или домашний каталог).
2. В терминале перейдите туда:

```bash
# macOS / Linux — пример:
cd ~

# Windows PowerShell — пример:
cd $HOME\Documents
```

3. Склонируйте репозиторий:

```bash
git clone https://github.com/Xpehutta/KatoPro.git
```

Что произойдёт:

- Git создаст папку `KatoPro`;
- в неё скачается весь код с GitHub;
- появится скрытая папка `.git` (нужна для обновлений через `git pull`).

4. Войдите в папку проекта:

```bash
cd KatoPro
```

5. Убедитесь, что вы на ветке `main` и видите файлы проекта:

```bash
git branch
git status
ls
```

Ожидаемо:

- ветка `main` (или `* main`);
- в списке файлов есть `docker-compose.yml`, `Dockerfile`, `README.md`, папки `src/`, `data/`.

> Альтернатива без Git: на странице репозитория нажмите **Code → Download ZIP**, распакуйте архив и выполните `cd` в эту папку.  
> Минус ZIP: позже сложнее обновляться — для обновлений удобнее именно `git clone` + `git pull`.

### Шаг 4. Обновление кода (git pull) — когда проект уже скачан

Если папка `KatoPro` у вас **уже есть**, не делайте `git clone` повторно. Вместо этого обновите код:

1. Перейдите в папку проекта:

```bash
cd путь/к/KatoPro
```

2. Получите последние изменения с GitHub:

```bash
git pull origin main
```

3. Если сервис уже был запущен раньше, пересоберите и перезапустите контейнер:

```bash
docker compose up -d --build
```

Кратко:

| Ситуация | Команда |
|----------|---------|
| Первый раз | `git clone https://github.com/Xpehutta/KatoPro.git` → `cd KatoPro` |
| Уже клонировали раньше | `cd KatoPro` → `git pull origin main` |
| После обновления кода | `docker compose up -d --build` |

Если `git pull` пишет про конфликты локальных правок — сохраните свои файлы из `data/` и `generated/` (они обычно не в Git), затем разберитесь с конфликтами или заново склонируйте в новую папку.

### Шаг 5. Подготовьте исходные Excel-файлы

1. Откройте папку `data/` внутри проекта.
2. Положите туда один или несколько файлов `.xlsx` торговых точек.
3. Удобный формат имени:

```text
МесОтчГГГГММ_НазваниеТочки.xlsx
```

Примеры:

```text
data/МесОтч202603_Смола.xlsx
data/МесОтч202603_Берёзка.xlsx
```

> Важно: в книге должен быть хотя бы один лист предыдущего месяца  
> (например, `Март26`), чтобы на его основе создать следующий (`Апрель26`).

На этом шаге достаточно просто положить файлы в `data/`.  
Точки можно будет добавить уже через веб-интерфейс.

Файлы в `data/` и `generated/` хранятся у вас локально и **не перезаписываются** обычным `git pull` (они в `.gitignore` / не обязательны для кода).

### Шаг 6. Запустите сервис

В терминале, находясь в папке `KatoPro`, выполните:

```bash
docker compose up -d --build
```

Что означает команда:

| Часть | Смысл |
|-------|--------|
| `docker compose` | управление сервисами проекта |
| `up` | создать и запустить контейнер |
| `-d` | работать в фоне (терминал останется свободным) |
| `--build` | собрать образ приложения |

Первый запуск может занять несколько минут (скачивание Python-образа и установка зависимостей).

Дождитесь сообщения вроде:

```text
Container katopro Started
```

Проверьте, что контейнер работает:

```bash
docker compose ps
```

В колонке `STATUS` должно быть `Up ...`.

Проверка «живости» сервиса:

```bash
curl http://localhost:8000/health
```

Ожидаемый ответ:

```json
{"status":"ok","message":"Сервис работает"}
```

### Шаг 7. Откройте веб-интерфейс

В браузере откройте:

[http://localhost:8000](http://localhost:8000)

Должна открыться русскоязычная страница KatoPro.

Полезные ссылки:

| Адрес | Назначение |
|-------|------------|
| http://localhost:8000 | основной интерфейс |
| http://localhost:8000/docs | документация API (Swagger) |
| http://localhost:8000/health | быстрая проверка, что сервис жив |

### Шаг 8. Добавьте торговые точки

На странице найдите блок **«1. Торговые точки»**.

**Способ A — загрузка файлов с компьютера:**

1. Нажмите выбор файлов и отметьте один или несколько `.xlsx`.
2. При загрузке нескольких файлов названия точек берутся из имён файлов  
   (`МесОтч202603_Смола.xlsx` → точка «Смола»).
3. Нажмите **«Загрузить выбранные»**.
4. Если файл с таким именем уже есть в `data/` — подтвердите замену  
   (или откажитесь: совпадающие файлы будут пропущены).

**Способ B — файлы уже лежат в `data/`:**

1. Нажмите **«Импорт всех из data/»**.
2. Убедитесь, что точки появились в списке ниже  
   (уже привязанные файлы пропускаются).

В блоке **«Файлы на диске»** можно:

- смотреть исходники (`data/`) и результаты (`generated/`);
- скачивать файлы;
- из вкладки **Сгенерированные** — **«В исходные»** (переместить файл в `data/`);
- удалять ненужные файлы.

### Шаг 9. Сгенерируйте отчёт на новый месяц

В блоке **«2. Параметры генерации»**:

1. Выберите **год** (например, `2026`).
2. Выберите **месяц** (например, `Май`).
3. Отметьте нужные точки  
   (или ничего не отмечайте — тогда обработаются все).
4. Нажмите **«Сгенерировать Excel»**.
5. Если в `data/` нет файла **предыдущего** периода для точки  
   (например, для мая нужен `МесОтч202604_Смола.xlsx`):
   - откроется диалог с рекомендацией периода;
   - можно скорректировать год/месяц **для каждой точки**;
   - или нажать «Всё равно продолжить».
6. Если файл за этот период уже есть в `generated/` — подтвердите пересоздание  
   или нажмите «Отмена» и выберите другой период/точку.
7. Дождитесь сообщения об успехе.
8. Скачайте файл:
   - по ссылке в результате генерации, или
   - во вкладке **«Сгенерированные»** блока «Файлы на диске».

Готовые файлы также появляются на диске в папке:

```text
generated/
```

Пример имени:

```text
generated/МесОтч202605_Смола.xlsx
```

### Шаг 10. Проверьте результат в Excel

1. Откройте скачанный файл в Excel / LibreOffice.
2. Найдите новый лист (например, `Май26`).
3. Убедитесь, что:
   - даты идут с 1-го по последнее число месяца;
   - выходные и праздники подсвечены;
   - формулы остатка и ТМ ссылаются на предыдущий месяц;
   - для ввода открыты нужные ячейки, остальное защищено.

Пароль защиты листа по умолчанию задан в `config.yaml`:

```yaml
protection_password: "secret"
```

### Шаг 11. Остановка, повторный запуск и обновление

Остановить сервис:

```bash
docker compose down
```

Запустить снова (без полной пересборки, если код не менялся):

```bash
docker compose up -d
```

Обновить код с GitHub и пересобрать:

```bash
cd путь/к/KatoPro
git pull origin main
docker compose up -d --build
```

Посмотреть логи, если что-то пошло не так:

```bash
docker compose logs -f
```

Выход из просмотра логов: `Ctrl + C`.

### Шпаргалка команд

```bash
# 1) Первый раз: скачать репозиторий
git clone https://github.com/Xpehutta/KatoPro.git
cd KatoPro

# 2) Положить Excel в data/

# 3) Запустить
docker compose up -d --build

# 4) Открыть http://localhost:8000

# --- позже: обновить проект ---
cd KatoPro
git pull origin main
docker compose up -d --build

# --- остановить ---
docker compose down
```

### Если что-то не работает

| Проблема | Что сделать |
|----------|-------------|
| `git: command not found` | Установите Git (шаг 2) и откройте новый терминал |
| `docker: command not found` | Установите/запустите Docker Desktop и откройте новый терминал |
| `fatal: destination path 'KatoPro' already exists` | Папка уже есть — зайдите в неё (`cd KatoPro`) и сделайте `git pull origin main` |
| `git pull` не проходит / конфликты | Сохраните `data/` и `generated/`, затем клонируйте заново в другую папку или разберите конфликт |
| Порт `8000` занят | Остановите другую программу на 8000 или измените порты в `docker-compose.yml` |
| Страница не открывается | Выполните `docker compose ps` и `docker compose logs -f` |
| «Исходный файл не найден» | Положите `.xlsx` в `data/` и снова загрузите/импортируйте точку |
| «Не найден лист предыдущего месяца» | Проверьте, что в книге есть листы вида `Март26`, `Апрель26` |
| Диалог про отсутствие файла предыдущего периода | Выберите рекомендованный месяц или загрузите `МесОтч…` за прошлый месяц в `data/` |
| Контейнер сразу падает | Смотрите логи: `docker compose logs --tail=100` |

### Запуск без Docker (альтернатива)

Если Docker использовать не хотите, нужен Python 3.10+:

```bash
cd KatoPro
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn src.main:app --host 0.0.0.0 --port 8000
```

Затем откройте [http://localhost:8000](http://localhost:8000).

## 3. Подготовка исходных файлов

Для каждой торговой точки нужен Excel-файл с историей месяцев  
(листы вида `Март26`, `Апрель26` и т.д.).

### Куда класть файлы

Положите их в папку:

```text
data/
```

### Как лучше называть файлы

Удобный шаблон:

```text
МесОтчГГГГММ_НазваниеТочки.xlsx
```

Примеры:

- `МесОтч202603_Смола.xlsx`
- `МесОтч202604_Берёзка.xlsx`

Если имя файла такое, программа сможет **автоматически подставить** название точки при загрузке.

## 4. Добавление торговых точек

Откройте главную страницу сервиса. Блок **«1. Торговые точки»**.

### Способ 1 — загрузить файлы (можно сразу несколько)

1. Нажмите выбор файлов и отметьте один или несколько `.xlsx`  
   (в диалоге можно выделить пачку через Ctrl/Cmd или Shift).
2. Для **одного** файла без имени в названии укажите название точки вручную.
3. Для **нескольких** файлов названия берутся из имён:  
   `МесОтч202603_Берёзка.xlsx` → «Берёзка».
4. Нажмите **«Загрузить выбранные»**.
5. Если в `data/` уже есть файл с **тем же именем** (период + точка),  
   сервис спросит согласие на замену:
   - **Да, заменить** — файлы перезапишутся;
   - **Отмена** — совпадающие имена будут пропущены, остальные загрузятся.

Кнопка **«Импорт всех из data/»** регистрирует Excel из папки `data/`, которые ещё не привязаны к точкам.

### Способ 2 — импорт из папки data/

Если файлы уже лежат в `data/`:

1. Нажмите **«Импорт всех из data/»**.
2. Уже привязанные файлы будут пропущены; новые точки появятся в списке.

### Просмотр файлов

В блоке **«Файлы на диске»**:

- вкладка **Исходные (data/)** — загруженные книги точек;
- вкладка **Сгенерированные** — готовые отчёты из `generated/`;
- кнопка **Обновить** — перечитать списки;
- **Удалить файлы** — выбор папок `data/` и/или `generated/` (файлы → `trash/`; при очистке `data/` список точек тоже очищается);
- **Скачать** — сохранить файл на компьютер;
- **В исходные** (только для сгенерированных) — переместить файл в `data/` и привязать точку;
- **Удалить** — после подтверждения файл перемещается в папку `trash/` (не удаляется навсегда).

При удалении исходного файла, привязанного к точке, точка автоматически убирается из списка.  
Файлы в корзине лежат в `trash/data/` или `trash/generated/` с меткой времени в имени.

### Удаление точки

В списке точек нажмите **«Удалить»**.  
Точка исчезнет из списка генерации; сам Excel в `data/` по умолчанию **не удаляется**.

Кнопка **«Обновить»** у списка точек анализирует `data/`: если есть файлы точек, которых ещё нет в списке, предложит их добавить.

> Список точек хранится в `data/points.yaml` и сохраняется между перезапусками.

## 5. Генерация отчёта

Блок **«2. Параметры генерации»**.

1. Выберите **год** и **месяц** нового отчёта.
2. Отметьте нужные торговые точки  
   (если ничего не отмечено — будут обработаны **все**).
3. Нажмите **«Сгенерировать Excel»**.
4. **Проверка предыдущего периода.**  
   Для генерации месяца N в `data/` должен быть файл месяца N−1  
   с именем вида `МесОтчГГГГММ_НазваниеТочки.xlsx`.  
   Если файла нет:
   - сервис покажет, какого файла не хватает;
   - предложит **рекомендуемый период** (следующий после последнего файла в `data/`);
   - позволит выбрать другой год/месяц **для каждой точки**;
   - кнопка «Всё равно продолжить» запускает генерацию без этой проверки.
5. Если отчёт за выбранный период уже есть в `generated/` — подтвердите пересоздание  
   или нажмите «Отмена» и выберите другой период/точку.
6. Дождитесь результата.
7. Скачайте файл по ссылке в результате или во вкладке **«Сгенерированные»** блока «Файлы на диске».

Готовые файлы также лежат на диске:

```text
generated/МесОтчГГГГММ_НазваниеТочки.xlsx
```

Пример: `generated/МесОтч202605_Смола.xlsx`.

## 6. Что проверить в новом файле

Откройте скачанный Excel и новый лист (например, `Май26`):

| Что проверить | Ожидание |
|--------------|----------|
| Даты в столбце A | С 1-го по последнее число месяца |
| Выходные / праздники | Подсвечены (как в ваших шаблонах) |
| Ячейка остатка / ТМ | Ссылки на предыдущий месяц, например `=Апрель26!I3` |
| Формулы | Суммы, средние, остаток в кассе |
| Защита листа | Редактируются только ячейки для ввода |
| Ввод данных | Выручка, возврат, карты, чеки, инкассация, продавцы, время |

Пароль защиты листа задаётся в `config.yaml` (`protection_password`).  
Сейчас по умолчанию: `secret`.

## 7. Полезные советы

1. **Генерируйте месяцы по порядку**  
   Сначала апрель, потом май — так корректнее связь с предыдущим листом.  
   Сервис сам предупредит, если в `data/` нет файла предыдущего периода.

2. **Для каждой точки — свой файл**  
   Не смешивайте разные точки в одной книге.

3. **В data/ должен быть файл предыдущего месяца**  
   Чтобы сделать май, желателен `МесОтч…04_Точка.xlsx` (апрель).  
   Рекомендуемый период в диалоге = следующий после последнего файла в `data/`.

4. **Праздники подтягиваются из интернета**  
   Если API недоступен, используются основные федеральные праздники (без части переносов).

5. **Можно добавлять сколько угодно точек**  
   Не только 13 — ограничение только вашими файлами и диском.

## 8. Типовые сценарии

### Сценарий A. Одна точка, следующий месяц

1. В `data/` лежит `МесОтч202603_Смола.xlsx` (последний лист — март).
2. Добавьте точку «Смола» (если ещё нет).
3. Сгенерируйте апрель 2026.
4. Скачайте файл из `generated/`.

### Сценарий B. Несколько новых точек

1. Загрузите через интерфейс сразу пачку `.xlsx` (или нажмите «Импорт всех из data/»).
2. Убедитесь, что точки появились в списке.
3. Выберите месяц и нажмите генерацию **без выбора точек** — обработаются все.
4. Скачайте каждый файл отдельно.

### Сценарий C. Обновить уже загруженный файл

1. Снова выберите тот же `.xlsx` (то же имя: период + точка).
2. Нажмите **«Загрузить выбранные»**.
3. В диалоге подтвердите замену файла.
4. Сгенерируйте нужный месяц.

### Сценарий D. Выбран слишком «ранний» следующий месяц

1. В `data/` есть только `МесОтч202604_Смола.xlsx` (апрель).
2. Вы выбрали генерацию **июня** 2026.
3. Сервис сообщит, что нет файла за май, и порекомендует генерировать **май**.
4. Подтвердите рекомендованный период (или скорректируйте вручную) и продолжите.

## 9. Частые вопросы

### Сервис не открывается на localhost:8000

- Проверьте, что контейнер запущен: `docker compose ps`
- Посмотрите логи: `docker compose logs -f`
- Убедитесь, что порт 8000 свободен

### Ошибка «Исходный файл не найден»

- Файл точки отсутствует в `data/`
- Или путь в списке точек указывает на старое имя файла  
  → добавьте точку заново / замените файл

### Ошибка «Не найден лист предыдущего месяца»

- В книге нет ни одного листа с именами вида `Март26`
- Проверьте, что листы названы по шаблону `МесяцГод`

### Диалог «Нет файла за предыдущий период»

- В `data/` нет `МесОтчГГГГММ_Точка.xlsx` за месяц перед выбранным
- Выберите рекомендованный период в диалоге или загрузите недостающий исходник
- «Всё равно продолжить» пропускает проверку (генерация может всё же найти лист внутри другой книги)

### Праздники выглядят неверно

- Проверьте интернет-доступ контейнера к API календаря
- При сбое используется встроенный резервный список

### Куда смотреть логи

```text
logs/
```

или:

```bash
docker compose logs -f katopro
```
