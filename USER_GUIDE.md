# Руководство пользователя KatoPro

Это пошаговая инструкция по работе с сервисом генерации ежемесячных отчётных книг Excel.

## 1. Что делает программа

KatoPro создаёт **чистый шаблон отчёта на новый месяц** для одной или нескольких торговых точек.

На основе уже заполненного файла за предыдущий месяц программа:

1. добавляет новый лист (например, `Май26`);
2. проставляет даты месяца;
3. выделяет выходные и праздничные дни РФ;
4. копирует формулы (выручка, остаток, суммы, средние);
5. связывает остаток и ТМ с предыдущим месяцем;
6. защищает лист, оставляя открытыми ячейки для ввода данных.

Почтовую рассылку сервис **не** выполняет — файлы сохраняются локально в папку `generated/`.

## 2. Как запустить

### Вариант A — Docker (рекомендуется)

1. Установите Docker Desktop (или Docker Engine + Compose).
2. Откройте папку проекта в терминале.
3. Выполните:

```bash
docker compose up -d --build
```

4. Откройте в браузере: [http://localhost:8000](http://localhost:8000)

Чтобы остановить:

```bash
docker compose down
```

### Вариант B — без Docker

```bash
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn src.main:app --host 0.0.0.0 --port 8000
```

Затем откройте [http://localhost:8000](http://localhost:8000).

## 3. Подготовка исходных файлов

Для каждой торговой точки нужен Excel-файл с историей месяцев  
(листы вида `Март26`, `Апрель26` и т.д.).

### Куда класть файлы

Положите их в папку:

```text
data/
```

### Как лучше называть файлы

Удобный шаблон:

```text
МесОтчГГГГММ_НазваниеТочки.xlsx
```

Примеры:

- `МесОтч202603_Смола.xlsx`
- `МесОтч202604_Берёзка.xlsx`

Если имя файла такое, программа сможет **автоматически подставить** название точки при загрузке.

## 4. Добавление торговых точек

Откройте главную страницу сервиса. Блок **«1. Торговые точки»**.

### Способ 1 — загрузить файл

1. Введите название точки (или оставьте пустым, если оно есть в имени файла).
2. Выберите `.xlsx`.
3. При необходимости включите «Заменить, если точка уже есть».
4. Нажмите **«Добавить из файла»**.

Файл сохранится в `data/`, точка появится в списке.

### Способ 2 — добавить вручную

Если файл уже лежит в `data/`:

1. Укажите название точки.
2. Выберите файл из выпадающего списка.
3. Нажмите **«Привязать файл»**.

### Удаление точки

В списке точек нажмите **«Удалить»**.  
Точка исчезнет из списка генерации; сам Excel в `data/` по умолчанию **не удаляется**.

> Список точек хранится в `data/points.yaml` и сохраняется между перезапусками.

## 5. Генерация отчёта

Блок **«2. Параметры генерации»**.

1. Выберите **год** и **месяц** нового отчёта.
2. Отметьте нужные торговые точки  
   (если ничего не отмечено — будут обработаны **все**).
3. Нажмите **«Сгенерировать Excel»**.
4. Дождитесь результата.
5. Скачайте файл по ссылке в результате или из блока «Недавно созданные файлы».

Готовые файлы также лежат на диске:

```text
generated/МесОтчГГГГММ_НазваниеТочки.xlsx
```

Пример: `generated/МесОтч202605_Смола.xlsx`.

## 6. Что проверить в новом файле

Откройте скачанный Excel и новый лист (например, `Май26`):

| Что проверить | Ожидание |
|--------------|----------|
| Даты в столбце A | С 1-го по последнее число месяца |
| Выходные / праздники | Подсвечены (как в ваших шаблонах) |
| Ячейка остатка / ТМ | Ссылки на предыдущий месяц, например `=Апрель26!I3` |
| Формулы | Суммы, средние, остаток в кассе |
| Защита листа | Редактируются только ячейки для ввода |
| Ввод данных | Выручка, возврат, карты, чеки, инкассация, продавцы, время |

Пароль защиты листа задаётся в `config.yaml` (`protection_password`).  
Сейчас по умолчанию: `secret`.

## 7. Полезные советы

1. **Генерируйте месяцы по порядку**  
   Сначала апрель, потом май — так корректнее связь с предыдущим листом.

2. **Для каждой точки — свой файл**  
   Не смешивайте разные точки в одной книге.

3. **Исходник должен содержать предыдущий месяц**  
   Чтобы сделать `Май26`, в книге желательно уже быть `Апрель26` (или ближайший более ранний месяц).

4. **Праздники подтягиваются из интернета**  
   Если API недоступен, используются основные федеральные праздники (без части переносов).

5. **Можно добавлять сколько угодно точек**  
   Не только 13 — ограничение только вашими файлами и диском.

## 8. Типовые сценарии

### Сценарий A. Одна точка, следующий месяц

1. В `data/` лежит `МесОтч202603_Смола.xlsx` (последний лист — март).
2. Добавьте точку «Смола» (если ещё нет).
3. Сгенерируйте апрель 2026.
4. Скачайте файл из `generated/`.

### Сценарий B. Несколько новых точек

1. Загрузите через интерфейс файлы всех точек.
2. Убедитесь, что они появились в списке.
3. Выберите месяц и нажмите генерацию **без** галочек — обработаются все.
4. Скачайте каждый файл отдельно.

### Сценарий C. Точка уже есть, нужно обновить файл

1. Загрузите новый `.xlsx`.
2. Укажите то же название.
3. Включите «Заменить, если точка уже есть».
4. Сгенерируйте нужный месяц.

## 9. Частые вопросы

### Сервис не открывается на localhost:8000

- Проверьте, что контейнер запущен: `docker compose ps`
- Посмотрите логи: `docker compose logs -f`
- Убедитесь, что порт 8000 свободен

### Ошибка «Исходный файл не найден»

- Файл точки отсутствует в `data/`
- Или путь в списке точек указывает на старое имя файла  
  → добавьте точку заново / замените файл

### Ошибка «Не найден лист предыдущего месяца»

- В книге нет ни одного листа с именами вида `Март26`
- Проверьте, что листы названы по шаблону `МесяцГод`

### Праздники выглядят неверно

- Проверьте интернет-доступ контейнера к API календаря
- При сбое используется встроенный резервный список

### Куда смотреть логи

```text
logs/
```

или:

```bash
docker compose logs -f katopro
```
