# Техническое задание (обновлённое) на разработку модуля автоматизации генерации ежемесячных отчётных книг Excel

## Введение

**Цель** – автоматизировать создание чистых шаблонов отчётных Excel-файлов для 13 торговых точек. На основе заполненного файла за предыдущий месяц генерируется новый лист для следующего месяца с корректными датами, выделением выходных и праздничных дней, формулами и защитой ячеек. Текущая версия **не включает** автоматическую почтовую рассылку; сгенерированные файлы сохраняются локально.

**Новые требования (по сравнению с первоначальной спецификацией):**

- Праздничные дни получаются автоматически из интернета (производственный календарь РФ) – без ручного ведения списка.
- Приложение поставляется в виде Docker-контейнера с возможностью запуска как REST API-сервиса (FastAPI) для удалённого вызова генерации.

---

## 1. Исходные данные

- Входной файл (заполненный продавцом) – например, `МесОтч202603_Смола.xlsx`.
- Для каждой торговой точки хранится отдельный Excel-файл с историей по месяцам (листы именуются по шаблону `МесяцГод`, например `Сентябрь25`).
- Список торговых точек (13 наименований) задаётся в конфигурации.
- Праздничные дни загружаются из внешнего API (см. раздел 4).

---

## 2. Требования к результату (без изменений)

Для каждого нового месяца `(year, month)` и каждой точки необходимо создать/обновить лист со следующими характеристиками:

- даты в столбце A (с 1‑го по последнее число месяца);
- выделение выходных и праздничных дней (цвет фона);
- все формулы (суммы, средние, остаток) – как в исходных шаблонах;
- начальный остаток и название ТМ ссылаются на предыдущий месяц (формулы);
- ячейки для ввода данных (выручка, возврат, оплата по картам, кол-во чеков, инкассация, продавцы/время) – **разблокированы**, остальные – **заблокированы** и защищены паролем (опционально).

## 3. Функциональные требования (с учётом новых пунктов)

### 3.1. Управление конфигурацией

Конфигурационный файл (`config.yaml`) содержит:

```yaml
points:
  - name: "Смола"
    file_path: "./data/МесОтч202603_Смола.xlsx"
  - name: "Берёзка"
    file_path: "./data/МесОтч202603_Берёзка.xlsx"
  # ... остальные точки

output_dir: "./generated"        # куда сохранять обновлённые файлы

protection_password: "secret"    # пароль для защиты листа (опционально)

# Настройки для получения производственного календаря
holidays_api:
  url: "https://production-calendar.ru/api/v1/calendar"
  # или другой источник, например, https://isdayoff.ru/api/
  year_field: "year"
  # параметры запроса – при необходимости
```

### 3.2. Получение праздничных дней из интернета

- При старте генерации для заданного года модуль должен выполнить HTTP-запрос к API производственного календаря РФ.
- Рекомендуемые источники:
  - `https://production-calendar.ru/api/v1/calendar?year=2026` (возвращает JSON со списком выходных и праздничных дней).
  - или `https://isdayoff.ru/api/getdata?year=2026` (возвращает строку с кодами дней).
- В ответе необходимо извлечь даты, которые являются **нерабочими праздничными днями** (не путать с перенесёнными выходными – нужны именно официальные праздники, чтобы правильно выделить их в календаре).
- Полученные даты кешируются на время выполнения сессии (можно сохранять в память или в локальный файл на сутки, чтобы не делать лишних запросов).
- Если API недоступен – использовать встроенный резервный список праздников (захардкоженный в коде на основе федеральных праздников, без учёта переносов). Это обеспечит устойчивость.

### 3.3. Основной алгоритм генерации (дополнен)

- При создании нового листа:
  - вызов API для получения праздников за текущий год;
  - формирование списка дат, попадающих в диапазон месяца (для использования в формулах и окраске);
  - обновление формулы `NETWORKDAYS` с учётом этих дат (диапазон праздников в `A15:A30` заполняется датами из полученного списка, отфильтрованными по месяцу).

### 3.4. REST API (для вызова извне)

Приложение должно предоставлять веб-интерфейс (FastAPI) с эндпоинтами:

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/generate` | Принимает JSON: `{"year": 2026, "month": 4, "points": ["Смола"]}` (если `points` не указан – обрабатываются все из конфига). Запускает генерацию для указанных точек и месяца. Возвращает статус и список обработанных файлов. |
| `GET` | `/status/{task_id}` | (опционально) для отслеживания длительных операций, если генерация будет асинхронной. |

- API должно быть документировано (Swagger UI будет доступен по `/docs`).

### 3.5. Docker-контейнеризация

- Приложение должно собираться в Docker-образ на основе `python:3.10-slim`.
- В состав образа включаются все зависимости, конфигурационный файл (как пример) и исходный код.
- При запуске контейнера стартует веб-сервер (например, через `uvicorn`).
- В `docker-compose.yml` должен быть описан сервис с монтированием томов для:
  - папки с данными (входные/выходные файлы);
  - конфигурации (чтобы можно было изменять без пересборки);
  - логов.
- Порт (например, `8000`) должен быть проброшен на хост.

## 9. Примечания по реализации получения праздников

Рекомендуемый API: **production-calendar.ru**  
Пример запроса: `GET https://production-calendar.ru/api/v1/calendar?year=2026`  
Ответ (сокращённо):
```json
{
  "holidays": [
    {"date": "2026-01-01", "name": "Новогодние каникулы"},
    {"date": "2026-01-02", ...},
    ...
  ],
  "preholidays": [...],
  "weekends": [...]
}
```
Необходимо извлечь все даты из поля `holidays` – они являются официальными нерабочими праздничными днями. Также можно объединить с `weekends`, но в РФ по умолчанию суббота и воскресенье – выходные, их выделять отдельно по календарю. В формуле `NETWORKDAYS` праздники указываются отдельно, поэтому достаточно передать только праздничные даты (не включая обычные субботы/воскресенья).

Альтернативный источник: `https://isdayoff.ru/api/getdata?year=2026` – возвращает строку из цифр (1 – выходной, 0 – рабочий) для каждого дня. Можно преобразовать в даты, где код = 1.

**Обработка ошибок:**  
Если API не отвечает (таймаут, ошибка сети) – используется встроенный список основных праздников (Новый год, 23 февраля, 8 марта, 1 мая, 9 мая, 12 июня, 4 ноября). Это не будет учитывать переносы, но позволит корректно выделить основные праздничные дни.
